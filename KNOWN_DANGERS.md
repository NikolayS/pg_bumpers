# Known dangers — operational hazards in pg_brakes' own setup. Honesty-first; see also [KNOWN_BYPASSES.md](KNOWN_BYPASSES.md).

## D1 — the strict `PUBLIC` lockdown (`deploy/sql/21_public_lockdown.sql`) can take down an existing production database — it is now OPT-IN, never the default
**FIXED in M2 (#108).** The original `deploy/sql/10_hardened_role.sql` revoked privileges from **`PUBLIC`** in schema `public` — `REVOKE EXECUTE ON ALL FUNCTIONS … FROM PUBLIC`, `ALTER DEFAULT PRIVILEGES … REVOKE EXECUTE … FROM PUBLIC`, and the `CREATE` / `TEMPORARY` / `lo_*` revokes. Those change the privilege model for **every role in the database, not just `pgb_agent`**, and applied to a production primary they stripped implicit `PUBLIC` grants the application relied on, breaking it for users (recoverable with a re-`GRANT`, but the blast radius is invisible until something breaks).

**What changed (#108).** The **default** BYO hardening `deploy/sql/10_hardened_role.sql` is now **agent-role-only** — it constrains `pgb_agent`/`pgb_applier` and **NEVER mutates `PUBLIC`**, so it is **safe to apply to an existing BYO database**. The strict `… FROM PUBLIC` revokes moved to a **separate, explicitly opt-in** file `deploy/sql/21_public_lockdown.sql` carrying its own ⚠️ header. The dev/test/CI fixture (`deploy/local-stack.sh`, `deploy/up.sh`, `deploy/test/wall_matrix.sh`) applies the lockdown so the strict posture stays tested; a BYO user does **not** apply it unless they choose to.

**If you opt into `21_public_lockdown.sql`:** apply it **ONLY** to a **dedicated/greenfield** database, OR rehearse it on a thin clone and smoke-test your application first (the M3 clone-rehearsal path). It is the DB-level belt-and-suspenders on top of the agent-only default; the honest residual it closes (the `PUBLIC`-default `lo_*`/`TEMP`/function-`EXECUTE` surfaces the agent retains on a shared DB) is documented in [KNOWN_BYPASSES.md](KNOWN_BYPASSES.md) (B-lo) and [docs/spec/SPEC.amendments.md](docs/spec/SPEC.amendments.md) (A-M2). On a shared BYO DB those residuals are gated, not by a global revoke, but — through the proxy (the realistic agent path) — by the M2a fail-closed read classifier (#114/#115: `SELECT lo_create()`/write-function/qualified-cast → `NotRead`, Blocked at the proxy floor) and — direct-to-DB — by the §3 network boundary.

The lesson: a safety tool's own setup must obey the tool's own floor — bounded, reversible, rehearsed-on-a-clone. The original default didn't; the agent-only default does, and the strict lockdown is now an explicit, blast-radius-checked, clone-rehearsed opt-in.
